1. Command Injection — False Positive / Mitigated in Current Flow

File / lines: iwa/blueprints/insecure/insecure_routes.py:80-86

@insecure_bp.route("/command_injection", methods = ["GET"])
def command_injection():
    arguments = request.args.get('arguments')
    home = os.getenv('APPHOME')
    cmd = home.join(INITCMD).join(arguments)
    os.system(cmd);

Issue: SAST reports this because user-controlled input appears to reach os.system(). However, in the current execution flow this is considered a false positive / mitigated finding: the command base value is not defined at runtime, so the route fails before command execution is reached.

Validation: The test request returned HTTP 500 and the proof file was not created, confirming that command execution was not reached in the current state.

Correction: Even though exploitation was not confirmed, the safer remediation is to remove os.system(). Remove shell command execution entirely and use safe Python APIs for the allowed actions.
# corrected implementation
@insecure_bp.route("/command_injection", methods=["GET"])
def command_injection():
    action = request.args.get("action", "status")
    if action == "status":
        load_1, load_5, load_15 = os.getloadavg()
        output = f"load average: {load_1:.2f}, {load_5:.2f}, {load_15:.2f}
"
    elif action == "disk":
        usage = shutil.disk_usage("/")
        output = f"total={usage.total} used={usage.used} free={usage.free}
"
    else:
        abort(400, "Unsupported action")
    return Response(output, mimetype="text/plain")

2. Dynamic Code Execution with exec()

File / lines: iwa/blueprints/insecure/insecure_routes.py:149-152

def source(script_path):
    with open(script_path, 'r') as script_file:
        script_contents = script_file.read()
        exec(script_contents)

Issue: The application reads a file and executes its content as Python code.

Correction: Remove exec(). If file viewing is required, read text only from an approved directory and return it as plain text.
# corrected implementation
@insecure_bp.route("/load_file", methods=["GET"])
def load_file():
    filename = request.args.get("filename", "")
    contents = source(filename)
    return Response(contents, mimetype="text/plain")

# source() validates that files stay inside safe_files/

3. Insecure Deserialization with pickle

File / lines: iwa/blueprints/auth/auth_routes.py:22, 86-94

import pickle

if 'rememberme' in request.cookies:
    b64 = request.cookies.get('rememberme')
    a = pickle.loads(base64.b64decode(b64))
    session.clear()
    session["email"] = a.email

Issue: A client-controlled cookie is deserialized with pickle.loads(). Pickle can execute code during deserialization.

Correction: Do not use pickle for client data. Use Flask signed sessions or a signed JSON token, for example with itsdangerous.
# corrected example
serializer = URLSafeSerializer(current_app.config["SECRET_KEY"],
                               salt="rememberme")
try:
    data = serializer.loads(request.cookies.get("rememberme"))
except BadSignature:
    abort(400, "Invalid remember-me token")

session.clear()
session["email"] = data["email"]
session["loggedin"] = True

4. SQL Injection

File / lines: iwa/blueprints/products/products_api_routes.py:31-38

@products_api_bp.route("/", methods=['GET'])
def search_products():
    keywords = request.args.get('keywords', '')
    db = get_db()
    query = "SELECT * FROM products WHERE name LIKE '%" + keywords + "%'"
    products = db.execute(query).fetchall()

Issue: User input is concatenated directly into a SQL query.

Correction: Use parameterized queries.
# corrected example
keywords = request.args.get("keywords", "")
pattern = f"%{keywords}%"
products = db.execute(
    "SELECT * FROM products WHERE name LIKE ?",
    (pattern,),
).fetchall()

5. Server-Side Template Injection / Unsafe Jinja2 Rendering

File / lines: iwa/blueprints/insecure/insecure_routes.py:96-101

template = request.form.get('template')
filename = request.form.get('filename')
t = Jinja2_Template(template)
name = source(filename)
html = t.render(name=name)
return html

Issue: User-controlled template content is rendered by Jinja2, which may allow template injection.

Correction: Never render user-supplied templates. Use static templates with render_template() and pass user input only as escaped data.
# corrected example
name = request.form.get("name", "")
return render_template("insecure/template_result.html", name=name)

6. Path Manipulation / Arbitrary File Path

File / lines: iwa/blueprints/insecure/insecure_routes.py:68-72

@insecure_bp.route("/load_file", methods = ["GET"])
def load_file():
    filename = request.args.get('filename')
    contents = source(filename)
    response = make_response(contents, 200)

Issue: A user-controlled filename is used directly without path validation.

Correction: Normalize the path and ensure it remains inside a safe base directory. Reject absolute paths and ../ traversal.
# corrected implementation
SAFE_FILES_DIR = (Path(__file__).resolve().parents[3] / "safe_files").resolve()

def source(script_path):
    requested = (SAFE_FILES_DIR / script_path).resolve()
    if not str(requested).startswith(str(SAFE_FILES_DIR)):
        abort(400, "Invalid path")
    if not requested.is_file():
        abort(404, "File not found")
    return requested.read_text(encoding="utf-8", errors="replace")

7. Insecure Cookie Attributes

File / lines: iwa/blueprints/insecure/insecure_routes.py:113-117

@insecure_bp.route("/insecure_cookies", methods=['POST'])
def insecure_cookies():
    session_id = request.form.get('session_id', '12345')
    response = make_response('Insecure cookie set')
    response.set_cookie('sessionid', session_id)

Issue: The cookie is created without hardened attributes.

Correction: Set secure attributes.
# corrected example
response.set_cookie(
    "sessionid",
    session_id,
    secure=True,
    httponly=True,
    samesite="Strict",
)

8. Hardcoded Secret Key

File / lines: main.py:24

os.environ["SESSION_SECRET"]="YourSessionSecret"

Issue: A secret value is hardcoded in source code.

Correction: Load the secret from the environment and fail safely if missing.
# corrected implementation in iwa/__init__.py
app.config.from_mapping(
    SECRET_KEY=os.environ.get("SESSION_SECRET"),
    DATABASE=os.path.join(app.instance_path, "insecurewebapp.sqlite"),
    SESSION_TYPE="filesystem",
)
if not app.config.get("SECRET_KEY"):
    raise RuntimeError("SESSION_SECRET environment variable is required")

9. Container Runs Without Non-Root User

File / lines: Dockerfile:1-17

FROM python:3.12-slim-bookworm
WORKDIR /usr/src/iwa
COPY requirements.txt /usr/src/iwa/requirements.txt
RUN pip3 install --no-cache-dir --upgrade -r requirements.txt
COPY . /usr/src/iwa/
CMD ["gunicorn", "--bind", "0.0.0.0", "--timeout", "600", "iwa:create_app()"]

Issue: No USER instruction is defined, so the application may run as root inside the container.

Correction: Create a non-root user and run the application with it.
# corrected example
RUN useradd -m appuser && chown -R appuser:appuser /usr/src/iwa
USER appuser
CMD ["gunicorn", "--bind", "0.0.0.0", "--timeout", "600", "iwa:create_app()"]

Conclusion

The project contains multiple vulnerability types detected by SAST and confirmed by code review. The remediation approach is to remove unsafe execution patterns, validate user-controlled inputs, use safe serialization and parameterized queries, harden cookies/secrets, and improve container security.
